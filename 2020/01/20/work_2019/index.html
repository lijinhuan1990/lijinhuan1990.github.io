数据下载

sra数据下载可能会因为存放路径原因而导致失败，需要更换命令(如采用以下命令或者是使用prefecth无法下载成功)

/data2/software/sratoolkit.2.9.2-centos_linux64/bin/fastq-dump --gzip SRR1170243
/data2/software/sratoolkit.2.9.2-centos_linux64/bin/fasterq-dump SRR1170243 -t /dev/shm -p

采用以下命令则可以下载成功：

wget -c https://sra-downloadb.be-md.ncbi.nlm.nih.gov/sos/sra-pub-run-1/SRR1170243/SRR1170243.1

软件issue说明：
https://github.com/ncbi/sra-tools/issues/214

2.homer 软件

1.参考基因组目录必须可写
2.默认物种是人的hg38参考基因组，如果是其他物种或者是人的另外版本，需要提供参考基因组fasta序列文件
3.输入文件为差异peak的bed文件，记录peak的染色体，起始位置，终止位置
4.命令
/public/homer/bin/findMotifsGenome.pl test.bed /data2//reference/rat/all.fa . -len 5,6,7,8,9,10,11,12 -rna -p 4 -size 400

或者 

/homer/bin/findMotifsGenome.pl all.txt hg38 . -len 5,6,7,8,9,10,11,12 -rna -p 4 -size 400

软件安装

昨天有小伙伴问我docker的问题，docker安装后可能会有损害root权限问题，所以集群或者是多人使用的小服务器应该是都不能安装了

很多软件的调配可以用conda，如果多软件存在调用冲突，可以创建不同的环境，以下是示例：

conda create -n cufflinks python=3
conda activate cufflinks
conda install -c bioconda cufflinks 
conda deactivate
